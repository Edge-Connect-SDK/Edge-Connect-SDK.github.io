          iOS #

          The Edge library is offered as a dynamic framework. The easiest way to install it is through Cocoapods.

          Requirements & Restrictions #

          Your deployment target has to be set to iOS 15 or higher.

          Installation #

          Cocoapods #

          The EdgeSDK has two dependencies to other private frameworks (Innolib and SignifyIndoorPositioning). To get access to the EdgeSDK and its dependencies, you need to have access to the Github organization ‘Edge-Connect-SDK’, and more specifically to the iOS repository in there: https://github.com/Edge-Connect-SDK/edge-sdk-ios .

          To install the EdgeSDK, first add this repository as a source to your Podfile. Also specify the main cocoapods spec repository since this is a requirement from Cocoapods when you add custom sources.

          source 'git@github.com:Edge-Connect-SDK/edge-sdk-ios.git'
source 'https://github.com/CocoaPods/Specs'

          Please contact your main contact at Edge to get access to this repository.

          Then in your Podfile, add the dependency to the EdgeSDK:

          pod 'EdgeSDK'
pod 'EdgeSDK-debug', :configurations => ['Debug']
pod 'EdgeSDK-release', :configurations => ['Release']

          Setup #

          After adding the framework to you project, youâ€™ll have to add a configuration file to your project with some settings to configure the project. This file should be named AppConfig.plist . You can override the files per target by adding a file named TargetConfig.plist. The values that can be defined in here are:

          NameTypeRequiredDefault valueExplanation
          WebAPIBAUStringtrue-The server is protected with basic auth. This property contains the username as a base64 encoded string.
          WebAPIBAPStringtrue-The server is protected with basic auth. This property contains the password as a base64 encoded string.
          EnvironmentStringtrue-Should be one of these values: development, alpha, acceptance or production. Used to determine which environment the app connects to.

          Then, make sure the following usage descriptions are filled in in your Info.plist (or InfoPlist.strings) file:

          • NSBluetoothAlwaysUsageDescription (Privacy - Bluetooth Always Usage Description)
          • NSCameraUsageDescription (Privacy - Camera Usage Description)
          • NSLocationWhenInUseUsageDescription (Privacy - Location When In Use Usage Description)

          To start the library, the create an AppCoordinator object and pass it the main window of your app, on which the library can show view controllers. Calling the start() function will return the initial view controller which the host app should display itself. A typical setup in the AppDelegate would look like this:

          window = UIWindow(frame: UIScreen.main.bounds)
coordinator = AppCoordinator(window: window!)
window?.rootViewController = coordinator.start()
window?.makeKeyAndVisible()

          DebugMode #

          When the DebugMode property is set to true in the config file, you can select a debug location instead of using your current location. The locations shown here match the buildings that are supported for your account.

          Overwrites #

          Fonts #

          Fonts can be overridden by adding the FontFileNames array and FontNames dictionary in your AppConfig (or TargetConfig) files. The FontFileNames is just a list of custom fonts you want to use in your app. The FontNames entry is a dictionary containing two more dictionaries: Base and Extra. Each of those contain another dictionary that maps a certain font (Base.Regular) to a certain font name. See the Styleguide to understand how the different fonts are used. An example config file that illustrates font overrides:

          <?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
	<key>FontFileNames</key>
	<array>
		<string>Edge-Bold.otf</string>
		<string>Edge-Book.otf</string>
		<string>Edge-Medium.otf</string>
		<string>P22MackinacPro-Bold-Italic.otf</string>
		<string>P22MackinacPro-Bold.otf</string>
		<string>P22MackinacPro-Medium.otf</string>
	</array>
	<key>FontNames</key>
	<dict>
		<key>Base</key>
		<dict>
			<key>Regular</key>
			<string>Edge-Book</string>
			<key>Bold</key>
			<string>Edge-Bold</string>
			<key>Black</key>
			<string>Edge-Medium</string>
		</dict>
		<key>Extra</key>
		<dict>
			<key>Black</key>
			<string>PMackinacProMedium</string>
			<key>Bold</key>
			<string>P22MackinacPro-Bold</string>
			<key>BoldItalic</key>
			<string>P22MackinacPro-BoldItalic</string>
		</dict>
	</dict>
</dict>
</plist>

          Colors #

          Colors can be overriden by adding a color catalog to the project and overwriting one or more of the colors that are mentioned in the Styleguide.

          Images #

          Images can be overriden by adding an image catalog to the project and overwriting one or more of the images that are mentioned in the Styleguide.
